Cómo usar diagramas Mermaid en Hugo

Guía completa para implementar y usar diagramas Mermaid en proyectos Hugo

🎨 Diagramas Mermaid en Hugo

Los diagramas Mermaid te permiten crear visualizaciones complejas usando solo texto. En este proyecto ya están configurados y listos para usar.

🚀 Cómo usar Mermaid

Para crear un diagrama, simplemente usa un bloque de código con el lenguaje mermaid:

MARKDOWN 
```mermaid
graph TD
    A[Inicio] --> B{Decisión}
    B -->|Sí| C[Acción 1]
    B -->|No| D[Acción 2]
```

📊 Tipos de diagramas disponibles

1. Diagramas de flujo

    graph TD
    A[Inicio] --> B{¿Condición?}
    B -->|Verdadero| C[Ejecutar acción]
    B -->|Falso| D[Otra acción]
    C --> E[Fin]
    D --> E

2. Diagramas de secuencia

    sequenceDiagram
    participant U as Usuario
    participant S as Sistema
    participant DB as Base de Datos
    
    U->>S: Iniciar sesión
    S->>DB: Validar credenciales
    DB-->>S: Resultado
    S-->>U: Respuesta de autenticación

3. Diagramas de clases

    classDiagram
    class Usuario {
        +String nombre
        +String email
        +iniciarSesion()
        +cerrarSesion()
    }
    
    class Administrador {
        +gestionarUsuarios()
        +generarReportes()
    }
    
    Usuario <|-- Administrador

4. Diagramas de entidad-relación

    erDiagram
    USUARIO ||--o{ PRESTAMO : "realiza"
    LIBRO ||--o{ PRESTAMO : "es_prestado_en"
    
    USUARIO {
        int id_usuario
        string nombre
        string email
        boolean administrador
    }
    
    LIBRO {
        int id_libro
        string titulo
        string autor
        int año
    }
    
    PRESTAMO {
        int id_prestamo
        date fecha_prestamo
        date fecha_devolucion
        int id_usuario
        int id_libro
    }

5. Diagramas de Gantt

    gantt
    title Cronograma del Proyecto
    dateFormat  YYYY-MM-DD
    section Fase 1
    Análisis        :done,    des1, 2024-01-01,2024-01-15
    Diseño          :done,    des2, 2024-01-16,2024-01-31
    section Fase 2
    Desarrollo      :active,  des3, 2024-02-01,2024-03-15
    Pruebas         :         des4, 2024-03-16,2024-03-31
    section Fase 3
    Despliegue      :         des5, 2024-04-01,2024-04-15

6. Diagramas de estado

    stateDiagram-v2
    [*] --> Inactivo
    Inactivo --> Activo : iniciar
    Activo --> Suspendido : suspender
    Suspendido --> Activo : reactivar
    Activo --> [*] : terminar
    Suspendido --> [*] : eliminar

🎨 Personalización y estilos

Puedes personalizar los colores y estilos de los diagramas:

    graph TD
    A[Nodo Normal] --> B[Nodo Destacado]
    A --> C[Nodo de Advertencia]
    A --> D[Nodo de Error]
    
    style B fill:#e1f5fe
    style C fill:#fff3e0
    style D fill:#ffebee

🔧 Configuración técnica

El soporte para Mermaid se implementó mediante:

  1. Render Hook: layouts/_default/_markup/render-codeblock-mermaid.html
  2. Script de carga: Agregado en layouts/_default/baseof.html
  3. CDN: Usando la versión ESM de Mermaid desde jsDelivr

Archivos modificados:

layouts/_default/_markup/render-codeblock-mermaid.html:

HTML 
<pre class="mermaid">
  {{ .Inner | htmlEscape | safeHTML }}
</pre>
{{ .Page.Store.Set "hasMermaid" true }}

layouts/_default/baseof.html (al final del body):

HTML 
{{ if .Store.Get "hasMermaid" }}
  <script type="module">
    import mermaid from 'https://cdn.jsdelivr.net/npm/mermaid/dist/mermaid.esm.min.mjs';
    mermaid.initialize({ startOnLoad: true });
  </script>
{{ end }}

💡 Consejos y mejores prácticas

  1. Mantén los diagramas simples: Evita diagramas demasiado complejos que sean difíciles de leer
  2. Usa etiquetas descriptivas: Nombres claros para nodos y conexiones
  3. Aprovecha los colores: Usa estilos para destacar elementos importantes
  4. Documenta diagramas complejos: Agrega explicaciones en texto cuando sea necesario
  5. Revisa la sintaxis: Usa la documentación oficial de Mermaid para verificar la sintaxis
